## 🧭 한서클럽포탈 Workflow

### 👤 비회원 (미가입 사용자)
- 🔍 등록된 동아리 목록 조회
- 📄 동아리 상세 정보 확인
- 🎯 모집 상태(모집전 / 모집중 / 모집완료)별 검색 기능 제공

> ※ 동아리 지원은 로그인 후 이용 가능합니다.

---

### 🙋‍♂️ 일반 사용자 (가입한 학생)
1. 📥 동아리 지원 폼 조회
2. ✍️ 폼 내용을 확인하고 각 문항에 답변 작성
3. 🚀 동아리 지원 요청 제출
4. 📋 내 지원 내역 및 처리 상태 확인

---

### 🧑‍💼 동아리 운영진
- 🛠️ 동아리 정보 수정 (관리자가 생성한 기본 정보 기반)
- 📊 지원자 목록 열람
- 🔍 각 지원자의 답변 상세 확인
- 📄 동아리 전용 지원 폼 생성/수정
- 🔄 지원자 상태 변경 (예: 접수 → 합격)
- 📈 동아리 대시보드로 통계 확인 (지원자 수 등)

---

### 🛡️ 관리자
- 👥 동아리 운영진 계정 생성
- 📝 동아리 글 작성과 동시에 신규 동아리 등록
- 📊 전체 동아리 대시보드 제공 (운영 현황 통계 등)

---

## 📝 동아리 지원폼 생성 워크플로우

운영진은 자신의 동아리에 대한 **지원폼 질문 목록**을 아래와 같은 형식으로 등록할 수 있습니다.

### ✅ 지원폼 생성 요청

`POST /api/clubadmin/clubs/{clubId}/form`

```json
{
  "labels": ["이름", "학과", "지원동기"]
}
```
🔁 서버 응답
```json
{
  "formId": 1
}
```
## 🗃️ 데이터 저장 구조

### 📌 1. `application_forms` 테이블

| id | club_id | updated_at                  |
|----|---------|-----------------------------|
| 1  | 1       | 2026-01-23 23:38:48.263955  |

- 새로운 지원폼이 생성되면 `club_id` 기준으로 1:1 매핑된 레코드가 생성됩니다.
- 하나의 동아리는 하나의 지원폼만 가질 수 있습니다.

---

### 📌 2. `form_questions` 테이블

| id | form_id | order_no | label     |
|----|---------|----------|-----------|
| 1  | 1       | 1        | 이름      |
| 2  | 1       | 2        | 학과      |
| 3  | 1       | 3        | 지원동기  |

- `form_id`를 기준으로 각 질문(label)이 순서대로 저장됩니다.
- 질문 순서는 `order_no` 필드로 제어됩니다.
